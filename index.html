Product essence

source: founder's words
The heart of the product — FIND ANY FILE IN SECONDS, without thinking about where it is.
Users stop keeping a folder structure in their head. Instead — one flat list + natural language search.

Founder quotes fact

  • "A desktop app monitors relevant folders" — an agent watches selected folders
  • "Files get a new name automatically following a clear structure" — auto-renaming
  • "Metadata of the files will be structurally enhanced by AI" — metadata enrichment
  • "All files are in only one / no folder (just a list)" — no-folder paradigm
  • "Searching will be possible based on keywords or natural language" — NLP search
  • "Searches can be saved showing like virtual folders" — saved queries as "folders"
  • "The design should be super minimalistic, but by using the app one should feel the power and convenience. Trustworthiness is critical."

Key principle interpretation

Minimalism ≠ emptiness. The client wants a sense of power and control within a very clean interface. Trustworthiness isn't just a word: a system that renames your files must give absolute confidence that nothing is broken.

Product boundaries fact

  • Not a cloud storage replacement (not a Google Drive alternative) — a smart agent on top of existing files
  • Hosting: European Secure Hosting Environment
  • No tech stack constraints at the design stage
  • Mobile app — out of MVP scope

Folder-based approach breaks down over time
The more files you have, the less logical the folder structure becomes. Hard to find what you need.

Default OS search doesn't understand context, doesn't support overlapping queries.

Files are scattered across different places
Downloads, Desktop, Documents, clouds — no single source of truth.
→ solution: monitored folders, unified dashboard
hypothesis

Fear that AI will break files
A product that automatically renames files causes anxiety. What if something disappears?
→ critical: trustworthiness, undo, AI action transparency
hypothesis

Conclusions & open questions

updated ongoing

"If / then / because" hypotheses hypothesis

Hypothesis 1 — Trustworthiness
If every AI action shows a preview + has global undo, then we remove the fear of "AI breaking my files," because control and reversibility are the foundation of trust in automated systems.
Hypothesis 2 — Onboarding
If onboarding = choosing the first folder + immediate result (files appear, renamed), then we overcome the "empty list" as risk #1 for adoption, because the first value must be visible within the first few minutes.
Hypothesis 3 — Minimalism vs power
If the dashboard = flat list + search bar + metadata columns (no folder sidebar), then we achieve the "minimalistic but powerful" feel, because Linear and Raycast have proven this combination is possible.

Kickoff questions ?

  • What happens if AI renamed a file incorrectly — is there undo? What does it look like?
  • Where are files physically stored after processing — in original folders or moved?
  • What is a "virtual folder" — a saved search or an actual OS folder?
  • Sharing — is it a public link or an invite to a specific person?
  • What does "trustworthy" look like in UI for you — what exactly should convey that feeling?
  • Are there screenshots or references you personally like — products with "powerful minimal"?
  • What first screen do you want to see after the 3-day trial?
