Суть продукту

джерело: слова клієнта
Серце продукту — ЗНАЙТИ БУДЬ-ЯКИЙ ФАЙЛ ЗА СЕКУНДИ, не думаючи де він.
Людина перестає тримати структуру папок у голові. Замість цього — один список + пошук природньою мовою.

Цитати засновника факт

  • "A desktop app monitors relevant folders" — агент стежить за обраними папками
  • "Files get a new name automatically following a clear structure" — авто-перейменування
  • "Metadata of the files will be structurally enhanced by AI" — збагачення метаданих
  • "All files are in only one / no folder (just a list)" — парадигма без папок
  • "Searching will be possible based on keywords or natural language" — NLP-пошук
  • "Searches can be saved showing like virtual folders" — збережені запити як "папки"
  • "The design should be super minimalistic, but by using the app one should feel the power and convenience. Trustworthiness is critical."

Принцип ключовий інтерпретація

Мінімалізм ≠ бідність. Клієнт хоче відчуття потужності і контролю в дуже чистому інтерфейсі. Trustworthiness — не просто слово: система, що перейменовує твої файли, повинна давати абсолютну впевненість що нічого не зіпсовано.

Межі продукту факт

  • Не хмарне сховище (не Google Drive замінник) — розумний агент поверх існуючих файлів
  • Hosting: European Secure Hosting Environment
  • Немає обмежень tech stack на дизайн-стадії
  • Mobile app — поза MVP

Больові точки з брифу факт

1
Folder-based підхід ламається з часом
Чим більше файлів — тим менш логічна структура папок. Важко знайти потрібне.
→ рішення: flat list + пошук замість ієрархії
факт
2
Пошук по файлах — повільний і неточний
Стандартний пошук ОС не розуміє контексту, не підтримує overlapping queries.
→ рішення: NLP search + збережені запити
факт
3
Ручне перейменування і тегування — нудна рутина
Люди або не роблять це взагалі, або витрачають час на організацію замість роботи.
→ рішення: AI auto-rename + auto-tagging
інтерпретація
4
Файли розкидані по різних місцях
Downloads, Desktop, Documents, хмари — нема одного place of truth.
→ рішення: моніторинг обраних папок, єдиний dashboard
гіпотеза
5
Страх що AI зіпсує файли
Продукт, який автоматично перейменовує — викликає тривогу. Що якщо щось зникне?
→ критично: trustworthiness, undo, прозорість дій AI
гіпотеза
📊
Додати: аналіз відгуків конкурентів (Hazel, Alfred, Everything, Dropbox, TagSpaces)
Формат: "Що бісить / Чому кидають / За що хвалять"

Бенчмарк

заповнити після research

Вимір 1 — Мінімалізм + відчуття потужності

Клієнт сказав: "super minimalistic, but feel the power." Це рідкісна комбінація — знайти референси де це зроблено добре.

Linear
Чистий список + keyboard shortcuts + відчуття швидкості. Benchmark для "powerful minimal".
Raycast
Command bar + flat list результатів. Search-first парадигма на desktop.
Notion
Inline AI дії з preview. Як показати що AI щось зробив, не лякаючи.

Вимір 2 — Trustworthiness при auto-actions

Продукт автоматично змінює файли. Як передати відчуття безпеки і контролю?

🛡️
Дослідити: як Grammarly, Figma, Hazel показують авто-дії і undo.
Шкала 1–5 за: прозорість / undo / preview перед дією / лог змін

Вимір 3 — Onboarding з порожнього стану

🚀
Дослідити: як Dropbox, Linear, Notion ведуть нового юзера до першої цінності.
Ризик №1: порожній список після встановлення — що показати?

Висновки та відкриті питання

оновлюється

Гіпотези "якщо / то / бо" гіпотеза

Гіпотеза 1 — Trustworthiness
Якщо кожна AI-дія показує preview + має global undo, то знімемо страх "AI зіпсує мої файли", бо контроль і зворотність — основа довіри до авто-систем.
Гіпотеза 2 — Onboarding
Якщо onboarding = вибір першої папки + миттєвий результат (файли з'явились, renamed), то подолаємо "пустий список" як ризик №1 adoption, бо перша цінність має бути видна в перші хвилини.
Гіпотеза 3 — Мінімалізм vs потужність
Якщо dashboard = flat list + search bar + metadata columns (без sidebar з папками), то досягнемо "minimalistic but powerful" відчуття, бо Linear і Raycast довели що це поєднання можливе.

Питання для kickoff ?

  • Що відбувається якщо AI перейменувало файл неправильно — є undo? Як це виглядає?
  • Де фізично зберігаються файли після обробки — в оригінальних папках чи переносяться?
  • Що таке "virtual folder" — збережений пошук чи справжня папка в ОС?
  • Sharing — це публічне посилання чи invite конкретної людини?
  • Що для тебе "trustworthy" виглядає в UI — що саме має передавати це відчуття?
  • Є скріни або референси які ти особисто любиш — продукти з "powerful minimal"?
  • Який перший екран ти хочеш побачити після 3-денного trial?
